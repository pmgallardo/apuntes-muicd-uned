### 1.2. Introducción a Cypher

#### 1.2.3. Codificación de consultas sencillas

EXAM=(2025J2.8,2025J2.10,2025J2.12)

La cláusula `MATCH` sirve para seleccionar elementos del grafo.

EXAM=(2023J2.9)

La sentencia `RETURN` devuelve un resultado. Si hay varias variables devueltas, las muestra en formato tabla.

Un **camino** (en inglés, *path*) en Neo4j/Cypher es una secuencia conectada de nodos y relaciones. Puede asignarse a una variable en un `MATCH` para devolverlo o manipularlo completo.

Dado este ejemplo:

```cypher
CREATE
  (ana:Persona {nombre: "Ana"}),
  (bea:Persona {nombre: "Bea"}),
  (carlos:Persona {nombre: "Carlos"}),
  (ana)-[:SIGUE]->(bea),
  (bea)-[:SIGUE]->(carlos);
```

La siguiente sentencia:

```cypher
MATCH camino = (:Persona {nombre: "Ana"})-[:SIGUE]->(:Persona)-[:SIGUE]->(:Persona)
RETURN camino;
```

Devolverá:

```text
Ana --SIGUE--> Bea --SIGUE--> Carlos
```

#### 4.3.2. Anidamiento de búsquedas

EXAM=(2023J2.10)

Si en una relación se añade asterisco al final del nombre, la relación se puede dar en tantos saltos como haga falta.

EXAM=(2023SO.13)

Cuando hay un número tras el asterisco, se limita el número de saltos. Se puede definir un rango empleando la sintaxis `[:RELACION*a..b]` (donde a y b son números positivos, y a < b).

```cypher
MATCH (u:Usuario {nombre: 'Carlos'})-[:SIGUE*1..2]->(d:Director)-[:DIRIGIO]->(p:Pelicula)
RETURN DISTINCT p.titulo AS Recomendaciones
```

### 4.4. Actualización de propiedades

#### 4.4.1. Modificación de las propiedades de nodos y relaciones

\

=EXAM(2024J2.7)

La función `SET` se emplea dentro de un `MATCH` para modificar propiedades de un nodo.

El operador `+` se puede emplear dentro de la función SET para añadir un elemento a una lista. Sin embargo, su implementación no es la más eficiente, por lo que se recomiento su sustitución por `ADD`.

```cypher
MATCH (j:Juego {titulo: 'Alone in the Dark'})
SET j.etiquetas = j.etiquetas + 'Aventura'
RETURN j.etiquetas
```

La cláusula `ADD` se emplea dentro de la función SET para añadir un elemento a una lista de manera eficiente.

```cypher
MATCH (j:Juego {titulo: 'Alone in the Dark'})
SET j.etiquetas ADD 'Aventura'
RETURN j.etiquetas
```

### 4.5

#### 4.5.3. Borrado en cascada

EXAM=(2025SO.11)

Para borrar todos los nodos y relaciones de una base de datos Neo4j se puede ejecutar la siguiente sentencia:

```cypher
MATCH (n) DETACH DELETE n;
```

Esta sentencia funciona porque separa los nodos y después los borra. Si se intentara borrar sin separar, es decir, sin la cláusula `DETACH`, daría error.
