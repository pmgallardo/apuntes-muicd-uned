### VD.EX.2022J2.C.1

#### Enunciado VD.EX.2022J2.C.1

¿Qué clase de variables se emplea para guardar categorías, etiquetas o nombres?

   a. Variables cuantitativas.  
   b. Variables discretas.  
   c. Variables nominales.  
   d. Variables deterministas.  

#### Solución VD.EX.2022J2.C.1

**Respuesta correcta: c. Las nominativas.**

Las variables que permiten almacenar clases, etiquetas o nombres pertenecen al grupo de las **variables cualitativas nominales**.

Según la clasificación habitual en representación de datos:

- Las variables **cuantitativas** representan cantidades numéricas medibles.
- Las variables **discretas** son un subtipo de cuantitativas.
- Las variables **deterministas** no constituyen una categoría relevante en la clasificación orientada a visualización.
- Las variables **nominales** almacenan categorías sin relación de orden.

Las variables nominales se caracterizan por:

- No tener jerarquía ni orden inherente.
- Permitir únicamente comparación por igualdad.
- Representar etiquetas como nombres, equipos o nacionalidades.

### VD.EX.2022J2.C.2

#### Enunciado VD.EX.2022J2.C.2

La creación de nuevas variables a partir de otras ya disponibles forma parte de la fase de:

a. Depuración o limpieza de datos.  
b. Filtrado o selección de variables y/o registros.  
c. Transformación de variables.  
d. Agrupamiento (clustering).  

#### Solución VD.EX.2022J2.C.2

**Respuesta correcta: c. Transformación de variables.**

La generación de nuevas variables a partir de variables existentes forma parte del proceso de **transformación de variables**, que se enmarca dentro del preprocesado o análisis de datos.

Esta tarea incluye:

- Creación de variables derivadas.
- Normalización o escalado.
- Discretización.
- Combinación de variables originales.

Por ejemplo, si a partir de las coordenadas $(x,y)$ generamos una nueva variable “distancia a portería”, estamos realizando una transformación.

Las otras opciones no son correctas porque:

- La limpieza de datos se centra en eliminar errores o inconsistencias.
- El filtrado implica selección, no creación.
- El clustering es una técnica de análisis no supervisado.

### VD.EX.2022J2.C.3

#### Enunciado VD.EX.2022J2.C.3

Cuando se afirma que el objetivo de la visualización debe quedar reflejado al construir un gráfico, ¿qué implica esto?

a. Que lo que no se quiere resaltar aparezca de la forma menos intuitiva posible.  
b. Que lo que se desea destacar se muestre de la manera más clara e intuitiva posible.  
c. Que lo que no se quiere resaltar se presente de la forma más intuitiva posible.  
d. Que lo que se desea destacar se represente de la forma menos intuitiva posible.  

#### Solución VD.EX.2022J2.C.3

**Respuesta correcta: b. A que una representación visual debe mostrar aquello que deseamos destacar de la manera más intuitiva posible.**

Uno de los principios fundamentales del diseño de visualizaciones es que el **objetivo comunicativo** debe guiar la codificación visual.

Esto implica:

- Identificar qué variable o patrón es relevante.
- Asignarle el canal visual más efectivo (posición, longitud, color, tamaño, etc.).
- Minimizar la carga cognitiva innecesaria.

Desde la perspectiva de la percepción visual y del visual mapping, debemos explotar los canales más precisos para aquello que queremos enfatizar.

Las demás opciones contradicen este principio al sugerir:

- Destacar lo irrelevante.
- Representar lo importante de forma poco intuitiva.

### VD.EX.2022J2.C.4

#### Enunciado VD.EX.2022J2.C.4

A partir de la representación gráfica proporcionada, indica cuál afirmación describe correctamente el gráfico:

![Representación gráfica](vd_ex_2022j2_c_4.png)

a. Se trata de un gráfico de columnas que facilita la comparación entre elementos, sin representar evolución temporal, y cuyo eje horizontal parte de cero para no distorsionar los valores.  
b. Es un gráfico de barras que permite comparar elementos y además sugiere evolución temporal, con el eje horizontal iniciando en cero para mantener la proporcionalidad.  
c. Es un gráfico de columnas que compara elementos y refleja evolución temporal, con el eje vertical comenzando en cero para respetar la escala.  
d. Es un gráfico de barras que compara elementos y muestra evolución temporal, con el eje vertical iniciando en cero para asegurar una lectura correcta.  

#### Solución VD.EX.2022J2.C.4

**Respuesta correcta: a.**

El gráfico mostrado es un **gráfico de barras apiladas**. Su objetivo principal es:

- Comparar varios elementos (Grupo 1, Grupo 2, etc.).
- Mostrar la **composición interna** de cada grupo (Aprobados / No aprobados).

Este tipo de representación permite visualizar simultáneamente:

- El total por grupo.
- La proporción relativa de cada componente dentro del grupo.

Sin embargo, aunque permite comparar composiciones, no es el mejor tipo de gráfico cuando se quiere comparar con precisión partes internas entre distintos grupos, debido a que solo el primer segmento comparte línea base común.

Las demás opciones no describen correctamente el objetivo principal del gráfico.

### VD.EX.2022J2.C.5

#### Enunciado VD.EX.2022J2.C.5

Respecto a técnicas de visualización estática, ¿cuál es la finalidad principal de un gráfico de barras apiladas como el ilustrado?

![Representación gráfica](vd_ex_2022j2_c_5.png)

a. Comparar múltiples categorías y, simultáneamente, evidenciar la composición interna de cada una.

c. No requiere una escala amplia para apreciar diferencias de tamaño entre grupos; por tanto, comparar   aprobados y no aprobados sería irrelevante.
d. Ninguna de las anteriores es correcta.  

#### Solución VD.EX.2022J2.C.5

**Respuesta correcta: a.**

- Comparar varias categorías.
- Mostrar simultáneamente la **composición interna** de cada categoría.

Este tipo de gráfico permite visualizar:

- El total de cada grupo.
- La proporción relativa de sus componentes.

Sin embargo, no es el más adecuado cuando se requiere comparar con precisión las partes entre grupos, debido a que solo la primera serie comparte línea base común.

### VD.EX.2022J2.E

#### Enunciado VD.EX.2022J2.E

Un medio deportivo desea elaborar una infografía que muestre, en función de la posición de los jugadores antes de disparar a portería, la probabilidad de marcar gol.

Los datos proceden de un repositorio estadístico perteneciente a una casa de apuestas y están organizados en una única tabla con los campos: fecha del encuentro, jugador, posición en el campo (delantero, defensa, centrocampista, etc.), equipo, coordenadas del disparo, minuto del gol y marcador final.

Ejemplo de registros:

| Fecha      | Jugador | Demarcación | Equipo | Coordenadas | Minuto | Resultado |
| ---------- | ------- | ----------- | ------ | ----------- | ------ | --------- |
| 12/02/2016 | 5       | Delantero   | B      | (100,250)   | 85     | 2-0       |
| 12/06/2016 | 15      | Defensa     | B      | (100,400)   | 15     | 2-0       |
| 15/02/2016 | 8       | Delantero   | R      | (201,314)   | 45     | 3-0       |

Cuestiones:

1. Indica qué tipo de preprocesamiento considerarías necesario a partir de los datos de ejemplo.

2. Clasifica cada variable según su tipo de dato.

3. Propón una visualización alineada con el objetivo descrito, justificando tu elección.

4. Opcional: realiza un boceto de la visualización usando los datos proporcionados.

#### Solución VD.EX.2022J2.E

**1. Procesamiento previo necesario**:

Antes de realizar la visualización es necesario un preprocesado orientado a aumentar la relación señal-ruido y a adaptar los datos al objetivo del estudio.
  
Las principales tareas serían:

- Limpieza de datos: detección de valores perdidos o inconsistentes en coordenadas, minuto o resultado.  
- Normalización del formato de fecha y separación estructurada del resultado final (goles a favor y en contra).  
- Descomposición de la variable coordenadas en dos variables numéricas: $x$ e $y$.  
- Generación de una variable binaria objetivo:  

$$
\text{Gol} =
\begin{cases}
1 & \text{si el tiro termina en gol} \\
0 & \text{si no termina en gol}
\end{cases}
$$

- Agregación espacial: discretización del campo en zonas o aplicación de una estimación de densidad para calcular la probabilidad de gol por región.  
- Cálculo de la probabilidad condicional:

$$
P(\text{Gol} \mid x,y) =
\frac{\text{Número de goles en la zona } (x,y)}
{\text{Número total de tiros en la zona } (x,y)}
$$

Este último paso es esencial para transformar los datos brutos en información interpretativa.

**2. Tipo de datos de cada variable**:

- Fecha: variable cuantitativa temporal (continua en dominio temporal).  
- Jugador: variable cualitativa nominal (identificador).  
- Demarcación: variable cualitativa nominal.  
- Equipo: variable cualitativa nominal.  
- Coordenadas: variable cuantitativa bidimensional continua $(x,y)$.  
- Minuto: variable cuantitativa discreta (ordinal temporal dentro del partido).  
- Resultado final: variable compuesta; puede descomponerse en dos cuantitativas discretas (goles local y visitante).

**3. Diseño de la visualización**:

El objetivo es representar la probabilidad de anotar un gol en función de la posición en el campo. Por tanto, la variable clave es espacial y continua en dos dimensiones.

La representación más adecuada sería un mapa de calor (heatmap) superpuesto sobre el esquema del campo de fútbol.

Justificación:

- La variable espacial $(x,y)$ se mapea directamente en posición horizontal y vertical del gráfico.  
- La probabilidad $P(\text{Gol} \mid x,y)$ se codifica mediante un canal visual de color, utilizando una escala secuencial (por ejemplo, de azul a rojo).  
- El uso del color explota la percepción preatentiva para identificar rápidamente zonas de alta peligrosidad.  
- Permite identificar patrones espaciales, como mayor probabilidad en zonas cercanas al área o centradas frente a portería.  

Opcionalmente, podrían incorporarse:

- Filtros por demarcación (para comparar delanteros vs defensas).  
- Facetado por equipo.  
- Segmentación temporal (primer tiempo vs segundo tiempo).

Desde el punto de vista del visual mapping:

- Datos espaciales → posición.  
- Probabilidad → intensidad de color.  
- Demarcación → posible codificación adicional mediante paneles o capas filtrables.  

Esta solución maximiza la correspondencia entre estructura matemática del problema (campo bidimensional continuo) y representación visual, respetando el principio de adecuación entre tipo de variable y canal visual.

**4. Esquema conceptual de la visualización**:

- Fondo: campo de fútbol simplificado.  
- Superposición: malla regular o estimación de densidad.  
- Escala lateral: barra de color indicando valores de probabilidad entre 0 y 1.  
- Título: “Probabilidad de gol según posición previa al tiro”.
